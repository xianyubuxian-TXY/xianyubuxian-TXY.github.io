# 1.引用限定符（Ref-qualifier）
引用限定符**限定的是调用者（this）的值类别！！**

## （1）语法
```cpp
class Foo {
    void bar() &;       // 只能被左值调用
    void bar() const&;  // 只能被 const 左值调用
    void bar() &&;      // 只能被右值调用
};
```

## （2）示例
```cpp
class MyString {
    std::string data_;
public:
    // 左值版本：返回引用，避免拷贝
    std::string& get() & { return data_; }
    
    // const 左值版本
    const std::string& get() const& { return data_; }
    
    // 右值版本：直接移动，对象即将销毁
    std::string get() && { return std::move(data_); }
};

MyString s;
s.get();                    // 调用 & 版本，返回 std::string&

const MyString cs;
cs.get();                   // 调用 const& 版本，返回 const std::string&

MyString().get();           // 调用 && 版本，返回 std::string（移动出来）
std::move(s).get();         // 调用 && 版本
```

## （3）为什么需要？
```cpp
// 没有引用限定符时的问题
class Bad {
    std::string data_;
public:
    std::string& get() { return data_; }
};

Bad().get() = "oops";  // 编译通过！但修改了临时对象，毫无意义
```
```cpp
// 使用引用限定符防止误用
class Good {
    std::string data_;
public:
    std::string& get() & { return data_; }  // 只允许左值调用
};

Good().get() = "oops";  // ❌ 编译错误！临时对象不能调用 & 方法
```

### <1>逻辑错误难以发现
```cpp
class Config {
    std::string value_;
public:
    std::string& get() { return value_; }
    Config withDefault() { return Config{"default"}; }
};

Config c;
c.withDefault().get() = "custom";  // 编译通过！但 c 没有任何改变

// 程序员以为修改了配置，实际上什么都没发生
// 这种 bug 可能潜伏很久
```

### <2>返回引用指向已销毁对象（真正危险）
```cpp
class Container {
    std::vector<int> data_{1, 2, 3};
public:
    std::vector<int>& data() { return data_; }
};

// 危险用法
auto& ref = Container().data();  // 临时对象立即销毁！
ref[0] = 42;                      // 💥 未定义行为！悬垂引用
```
**加上引用限定符后**
```cpp
class Container {
    std::vector<int> data_{1, 2, 3};
public:
    std::vector<int>& data() & { return data_; }        // 左值返回引用
    std::vector<int> data() && { return std::move(data_); }  // 右值返回值
};

auto& ref = Container().data();  // ❌ 编译错误！&& 版本返回的是值，不能绑定到 auto&
auto val = Container().data();   // ✅ 正确，移动出来
```

### <3> 实际踩坑场景
```cpp
class Builder {
    std::string result_;
public:
    Builder& append(const std::string& s) { 
        result_ += s; 
        return *this; 
    }
    std::string& str() { return result_; }
};

// 链式调用的坑
std::string& s = Builder().append("a").append("b").str();  
// s 是悬垂引用！Builder 临时对象已销毁

std::cout << s;  // 💥 未定义行为
```

## （4）引用限定符使用总结
![](https://cdn.jsdelivr.net/gh/xianyubuxian-TXY/hexo-images/images/20260119160238018.png)
```cpp
class Example {
    std::vector<int> data_;  // 大对象
    int count_ = 0;          // 小对象
    
public:
    // 大对象：区分左右值
    std::vector<int>& data() & { return data_; }
    const std::vector<int>& data() const& { return data_; }
    std::vector<int> data() && { return std::move(data_); }
    
    // 小对象：不区分，直接返回拷贝
    int count() const { return count_; }
};
```
