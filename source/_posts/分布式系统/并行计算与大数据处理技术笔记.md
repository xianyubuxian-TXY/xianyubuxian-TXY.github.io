### （2）存储访问体系结构
![](https://cdn.jsdelivr.net/gh/xianyubuxian-TXY/hexo-images/images/20260317094207253.png)
```bash
① 共享存储器 (Shared Memory)
┌─────────────────────────────────────────┐
│  CPU0    CPU1    CPU2    CPU3            │
│   │       │       │       │             │
│   └───────┴───┬───┴───────┘             │
│               │                         │
│          ┌────┴────┐                    │
│          │ 共享内存  │  ← 所有CPU看到同一块内存 │
│          └─────────┘                    │
└─────────────────────────────────────────┘


② 分布存储器 (Distributed Memory)
┌────────────┐    网络    ┌────────────┐
│ CPU0       │ ←──────→  │ CPU1       │
│ 本地内存0   │            │ 本地内存1   │
└────────────┘            └────────────┘
       ↕ 网络                  ↕ 网络
┌────────────┐            ┌────────────┐
│ CPU2       │ ←──────→  │ CPU3       │
│ 本地内存2   │            │ 本地内存3   │
└────────────┘            └────────────┘
各CPU只能直接访问自己的内存，访问别人的要"发消息"


③ 分布共享存储 (Distributed Shared Memory)
┌────────────────┐    网络    ┌────────────────┐
│ CPU0           │ ←──────→  │ CPU1           │
│ 本地内存0+Cache │            │ 本地内存1+Cache │
└────────────────┘            └────────────────┘
  物理上分布，但逻辑上所有CPU看到"统一的地址空间"
  通过 Cache 一致性协议 维护数据正确性
```
![](https://cdn.jsdelivr.net/gh/xianyubuxian-TXY/hexo-images/images/20260317094647003.png)
- 统一编址，物理分布：访问本地存储快，访问其它存储慢，但对于使用者是透明的，并不知道访问的是本地还是其它 ——> 每个存储都有缓存，如果本地访问了其它存储并修改，会在本地缓存存储新值，但在其它缓存可能仍存的是旧值，这就导致了“**缓存一致性问题**”

### （2）伪代码实现
```java
import java.io.IOException;
import java.util.StringTokenizer;
import org.apache.hadoop.io.Text;
import org.apache.hadoop.mapreduce.Mapper;

// Mapper<KEYIN, VALUEIN, KEYOUT, VALUEOUT>
// 这里表示：
// 输入的 key 类型是 Text
// 输入的 value 类型是 Text
// 输出的 key 类型是 Text
// 输出的 value 类型是 Text
public class InvertedIndexMapper extends Mapper<Text, Text, Text, Text> {

    @Override
    protected void map(Text key, Text value, Context context)
            throws IOException, InterruptedException {

        // 1. 获取当前正在处理的文件切片 FileSplit
        // context.getInputSplit() 能拿到当前 MapTask 正在处理的输入分片
        FileSplit fileSplit = (FileSplit) context.getInputSplit();

        // 2. 从 FileSplit 中取出当前文件名
        // 例如：a.txt
        String fileName = fileSplit.getPath().getName();

        // 3. 定义一个 Text 对象 word，用来保存切分出来的单词
        Text word = new Text();

        // 4. 构造输出 value：文件名#偏移量
        // key 在这里表示当前这一行在文件中的偏移量（line offset）
        // 例如：a.txt#128
        Text fileName_lineOffset = new Text(fileName + "#" + key.toString());

        // 5. 对当前行内容进行分词
        // value.toString() 是当前这一行文本内容
        // StringTokenizer 默认按空白符切分
        StringTokenizer itr = new StringTokenizer(value.toString());

        // 6. 遍历这一行中的所有单词
        for (; itr.hasMoreTokens(); ) {

            // 7. 取出一个单词，赋给 word
            word.set(itr.nextToken());

            // 8. 输出键值对
            // key   = 单词
            // value = 文件名#行偏移量
            // 例如输出：hello -> a.txt#128
            context.write(word, fileName_lineOffset);
        }
    }
}
```
```java
import java.io.IOException;
import java.util.Collections;   // 这里其实没有使用到，属于多余导包
import java.util.Iterator;      // 用于遍历 Iterable<Text> values 中的每个 value
import org.apache.hadoop.io.Text;
import org.apache.hadoop.mapreduce.Reducer;

// Reducer<KEYIN, VALUEIN, KEYOUT, VALUEOUT>
// 这里表示：
// reduce阶段输入的 key   类型是 Text
// reduce阶段输入的 value 类型是 Text
// reduce阶段输出的 key   类型是 Text
// reduce阶段输出的 value 类型是 Text
public class InvertedIndexReducer extends Reducer<Text, Text, Text, Text>
{
    @Override
    protected void reduce(Text key, Iterable<Text> values, Context context)
            throws IOException, InterruptedException
    {
        // values 是同一个 key 对应的所有 value 的集合
        // 例如：
        // key = "fish"
        // values = ["doc1#0", "doc1#8", "doc2#0", "doc2#8"]

        // 获取 values 的迭代器，后面通过 it.hasNext() 和 it.next() 逐个遍历
        Iterator<Text> it = values.iterator();

        // 使用 StringBuilder 拼接最终输出字符串
        // 例如把多个位置拼成：
        // "doc1#0;doc1#8;doc2#0;doc2#8"
        StringBuilder all = new StringBuilder();

        // 先判断 values 中是否至少有一个元素
        // 这样做的目的是：
        // 1. 先把第一个元素直接放进去
        // 2. 后面再在每个元素前面补 ";"
        // 这样就不会出现开头多一个分号的问题
        if(it.hasNext())
            all.append(it.next().toString());

        // 继续处理剩余元素
        for(; it.hasNext(); )
        {
            // 先拼接分隔符 ";"
            all.append(";");

            // 再拼接下一个 value
            all.append(it.next().toString());
        }

        // 将最终拼接结果作为 value 输出
        // 输出结果形如：
        // ("fish", "doc1#0;doc1#8;doc2#0;doc2#8")
        context.write(key, new Text(all.toString()));
    }

    // 最终输出键值对示例：
    // ("fish", "doc1#0;doc1#8;doc2#0;doc2#8")
}
```
```java
public class InvertedIndexer
{
    public static void main(String[] args)
    {
        try {
            // 1. 创建 Hadoop 配置信息对象
            // Configuration 中保存了 Hadoop 运行时需要的各种配置信息，
            // 例如 HDFS 地址、MapReduce 框架配置等
            Configuration conf = new Configuration();

            // 2. 创建一个 Job 对象
            // 这个 Job 就代表一个完整的 MapReduce 任务
            // "invert index" 是这个任务的名字，方便在控制台或 Web UI 中查看
            Job job = new Job(conf, "invert index");

            // 3. 指定当前 Job 所在的主类
            // Hadoop 会根据这个类找到对应的 jar 包并提交到集群运行
            job.setJarByClass(InvertedIndexer.class);

            // 4. 设置输入格式类
            // TextInputFormat 表示按文本方式读取输入文件
            // 默认按行读取：
            // key   = 当前行的起始偏移量
            // value = 当前行内容
            job.setInputFormatClass(TextInputFormat.class);

            // 5. 设置 Mapper 类
            // 指定 Map 阶段由哪个类来处理输入数据
            job.setMapperClass(InvertedIndexMapper.class);

            // 6. 设置 Reducer 类
            // 指定 Reduce 阶段由哪个类来汇总 Mapper 输出的中间结果
            job.setReducerClass(InvertedIndexReducer.class);

            // 7. 设置最终输出的 key 类型
            // 倒排索引最终输出 key 是单词，因此类型是 Text
            job.setOutputKeyClass(Text.class);

            // 8. 设置最终输出的 value 类型
            // 倒排索引最终输出 value 是该单词出现的位置列表，因此类型也是 Text
            job.setOutputValueClass(Text.class);

            // 9. 设置输入路径
            // args[0] 表示程序运行时传入的第一个参数，一般是输入目录
            // 例如：/input
            FileInputFormat.addInputPath(job, new Path(args[0]));

            // 10. 设置输出路径
            // args[1] 表示程序运行时传入的第二个参数，一般是输出目录
            // 例如：/output
            FileOutputFormat.setOutputPath(job, new Path(args[1]));

            // 11. 提交任务并等待任务完成
            // waitForCompletion(true) 表示提交任务后一直等待直到运行结束
            // true 表示在控制台显示任务进度信息
            //
            // 如果任务成功完成，返回 true，程序退出码为 0
            // 如果任务失败，返回 false，程序退出码为 1
            System.exit(job.waitForCompletion(true) ? 0 : 1);

        } catch (Exception e) {
            // 12. 如果程序运行过程中出现异常，则打印异常堆栈信息
            e.printStackTrace();
        }
    }
}
```

#### <2>RAID的实现方式
```bash
① 硬件 RAID（RAID 卡）
  ┌──────────┐
  │ RAID 控制卡│  ← 专用硬件芯片，有独立 CPU 和缓存
  │ (带电池)   │  ← 电池保护：突然断电时缓存数据不丢
  └─────┬────┘
        │
  ┌─────┼─────┬─────┬─────┐
  │硬盘0│硬盘1│硬盘2│硬盘3│
  └─────┴─────┴─────┴─────┘

  优点：性能最好，对操作系统透明（OS 只看到一块逻辑盘）
  缺点：贵（几千到几万元），RAID 卡坏了要同型号替换

  品牌：LSI (Broadcom)、Dell PERC、HP Smart Array


② 软件 RAID（操作系统实现）
  Linux：mdadm 命令
  Windows：磁盘管理 → 动态磁盘

  优点：免费，灵活
  缺点：占 CPU 资源，性能不如硬件 RAID
```
