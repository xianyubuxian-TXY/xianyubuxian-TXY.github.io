# 一、字符串
## 1.KMP算法
### （1）next数组
- 约定下标从1开始
- next[i]：表示“模式串”的第i个字符发生不匹配时，应该回溯到的位置
- next[i] = S[1...i-1]的最长相等前后缀长度+1
	- 串的前缀：包含第一个字符，且不包含最后一个字符的子串
	- 串的后缀：不包含最后一个字符，且包含第一个字符的子串
	- 记忆： ab ——> b的前缀是a吗？不对，为空

# 四、图

## （1）图的实现
```bash
设 V = 顶点数，E = 边数，deg(u) = 顶点 u 的度

┌───────────────────────┬──────────────┬──────────────┐
│ 操作                   │ 邻接矩阵      │ 邻接表        │
├───────────────────────┼──────────────┼──────────────┤
│ 查 u→v 是否有边        │ O(1) ✅       │ O(deg(u)) ❌  │
│ 遍历 u 的所有邻居      │ O(V) ❌       │ O(deg(u)) ✅  │
│ 遍历所有边             │ O(V²) ❌      │ O(V + E) ✅   │
│ 添加一条边             │ O(1) ✅       │ O(1) ✅       │
│ 删除一条边             │ O(1) ✅       │ O(deg(u)) ❌  │
│ 添加一个顶点           │ O(V²) 重建 ❌ │ O(1) ✅       │
│ 空间                   │ O(V²)        │ O(V + E)     │
└───────────────────────┴──────────────┴──────────────┘
```

## （3）最小生成树（Kruskal算法）
- **每次迭代，选出一条具有最小权值，且两端点不在同一连通分量上的边，加入生成树**
- **利用“最小堆”+“并查集”实现Kruskal算法**
	- **最小堆**：存储所有边，方便获取最小边
	- **并查集**：方便判断加入新边时，该边的两个顶点是否在同一个连通分量上

# 六、key-value存储原理
```bash
┌─────────────────────┬──────────────┬──────────────────────────────┐
│ 数据结构              │ 查找复杂度    │ 典型应用                      │
├─────────────────────┼──────────────┼──────────────────────────────┤
│ ① Hash Table        │ O(1)         │ Redis, Memcached, 各种内存KV  │
│ ② B+ Tree           │ O(log n)     │ MySQL InnoDB, PostgreSQL     │
│ ③ B Tree            │ O(log n)     │ MongoDB (WiredTiger), BoltDB │
│ ④ 红黑树（Red-Black） │ O(log n)     │ Java TreeMap, C++ std::map   │
│ ⑤ 跳表（Skip List）  │ O(log n)     │ Redis ZSet, LevelDB MemTable │
│ ⑥ LSM Tree          │ O(log n)~多层 │ LevelDB, RocksDB, Cassandra  │
│ ⑦ Trie / 前缀树      │ O(key长度)   │ 某些路由表, etcd (radix tree) │
│ ⑧ AVL Tree          │ O(log n)     │ 很少直接用于生产KV系统 ⚠️     │
└─────────────────────┴──────────────┴──────────────────────────────┘
```
