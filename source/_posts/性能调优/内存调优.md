## 2.Linux下可执行目标文件运行时的存储器映像
### （1）Linux 进程虚拟地址空间布局（从高到低）
```cpp
高地址 0xFFFFFFFF (32位) / 0x7FFFFFFFFFFF (64位)
┌─────────────────────────────┐
│         内核空间             │  ← 用户不可访问
├─────────────────────────────┤ 
│          栈 (Stack)         │  ← 向下增长 ↓
│             ↓               │     局部变量、函数参数、返回地址
│            ...              │
├─────────────────────────────┤
│                             │
│      （未分配区域）           │
│                             │
├─────────────────────────────┤
│             ↑               │
│          堆 (Heap)          │  ← 向上增长 ↑
│                             │     malloc/new 分配
├─────────────────────────────┤
│                             │
│      可读写数据段            │  ← .data（已初始化全局/静态变量）
│      (.data + .bss)         │     .bss（未初始化全局/静态变量）
│                             │
├─────────────────────────────┤
│                             │
│        只读段                │  ← .rodata（只读数据、字符串常量）
│    (.text + .rodata)        │     .text（代码段、机器指令）
│                             │
├─────────────────────────────┤
│         保留区               │  ← 防止空指针访问
└─────────────────────────────┘
低地址 0x00000000
```
