3.最长回文子串

发表于 2026-02-12 分类于 编程题 ， 动态规划 ， 字符串 Waline： 本文字数： 941 阅读时长 ≈ 3 分钟

1.题目描述

2.题目解析

  • 回文串的特点：
    • 关于中心（1个字符/2个字符）对称，故以每个字符为中心，使用双指针即可
  • 注意：
    • 必须分别以“1个字符”、“2个字符”为中心，向两侧扩展

3.代码实现

（1）要求版

  • 暴力解法即可：空间复杂度O(1),时间复杂度O(n^2)

2.进阶版

  • Manacher 算法：O(n) 时间，O(n) 空间，直接返回最长回文子串长度。

（1）Manacher 思路（核心点）

-把原串 A 变换成插入分隔符的新串 T，例如：
- A = “abba”
- T = “^#a#b#b#a#$”
- 这样所有回文都变成奇数长度（中心扩展统一处理）。

  • 定义 dp[i]：以 i 为中心的回文半径（在 T 中能扩展的长度，不含自身）。
  • 维护当前已知的最右回文边界 R 和其中心 C，用镜像性质加速。
  • 最终答案：max(dp[i])（它刚好等于原串中的最长回文长度）。

<1>为什么要“变换字符串”

原串里回文分两类：

  • 奇数回文：aba（中心是一个字符）
  • 偶数回文：abba（中心在两个字符之间）
    为了统一处理，我们把原串 s 变成新串 T：在每个字符之间插入 #，并加上两个哨兵 ^ 和 $ 防越界。
    例如：s = “abba”

这样做的好处：

  • 所有回文在 T 中都变成奇数长度（中心永远落在某个字符上，可能是原字符也可能是 #）
  • 扩展时不用分别处理奇偶了

<2>dp[i] 的含义是什么

定义数组 dp[i]：

  • dp[i] = 以 T[i] 为中心，向左右能扩展的“半径长度”（匹配的字符对数），不包含中心本身。

比如在 T=”^#a#b#b#a#$” 中，中心在中间 # 时可以扩展很远。
关键结论：

  • max(dp[i]) 刚好等于原串最长回文子串长度（因为 # 把间距填平了）

<3>center 和 right 是什么（Manacher 的“加速器”）

Manacher 用两个变量维护“当前扫到的最右回文”：

  • center：当前这个“最右”回文的中心（
  • right：这个回文能到达的最右边界，通常用 right = center + dp[center]
    当我们来到一个新位置 i 时（从左到右依次的每个字符），如果 i < right，说明它落在当前最右回文内部，可以利用“对称性”加速：
  • mir = 2*center - i 是 i 关于 C 的镜像点
  • 回文对称 ⇒ dp[i] 至少能继承 dp[mir] 的一部分，但不能超过右边界 R，所以：
    • p[i]=min(R−i,p[mir])
  • 然后再从这个初值继续尝试扩展（通常扩展次数很少，总体摊还 O(n)）。

<4>为什么是 O(n)

  • 看起来每个 i 都可能 while 扩展很多次，但注意：
    • R 只会单调向右移动
    • 每次 while 扩展成功，都会推动某个中心的 R 变大
    • 整体扩展的总次数不超过 T 的长度量级
      因此 总复杂度 O(n)。
