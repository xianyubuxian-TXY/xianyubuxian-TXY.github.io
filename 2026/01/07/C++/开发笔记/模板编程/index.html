2.类型萃取（Type Traits） 工具

1.概述

(1)头文件： #include<type_traits>
(2)功能：提供的编译期类型判断 / 转换工具，核心作用是在模板编程中获取或修改类型的属性，实现编译期决策。

2.类型判断类（判断类型属性）

  • 这类萃取工具以 is_xxx 命名，C++17 后提供 is_xxx_v 便捷别名（等价于 is_xxx::value），返回编译期布尔常量。

3.std::conditional_t<Condition, T, F> ：编译期条件类型选择工具

1.核心作用

  • 根据一个编译期布尔常量，在两种类型（T 和 F）中选择其一
  • 整个选择过程完全发生在编译阶段，不会产生任何运行时开销。
  • 如果 Condition 为 true → 最终类型是 T
  • 如果 Condition 为 false → 最终类型是 F

2.核心要求

  • Condition 必须是编译期可确定的布尔值
    • true/false 字面量
    • constexpr 常量
    • static_assert
    • is_same_v、is_integral_v…
    • 等类型萃取的结果，不能是运行时才能确定的变量。

4.可调用对象相关类

用于推导可调用对象（函数、Lambda、仿函数）的属性，是泛型回调、函数包装的核心工具。

(1)声明一个“空”主模板（primary template）

不给它任何定义

(2)对受支持的类型做显式特化（explicit specialization）

在特化里定义一个 using type = …;

  • 这样，connection_config::type 就是 MySQLConfig，而 connection_config::type 就是 RedisConfig。

注意：必须使用 typename 告诉编译器connection_config::type是“类型”，否则编译器无法知道它是“类型”还是“变量”

  • 以后在代码里直接写 Config_t，就等同于 connection_config::type，更简洁。

(4)为什么这么做？

1.集中管理支持列表：所有支持的连接类型映射都集中在 connection_config<…> 的特化里，新加一种连接只要加一个特化即可，其他逻辑不用改。
2.这种模式在 C++ 标准库里也很常见：比如 std::iterator_traits、std::char_traits、std::common_type<T,U>、std::hash 等，都是“空主模板 + 针对受支持类型的特化 + alias template” 组合的变体。
